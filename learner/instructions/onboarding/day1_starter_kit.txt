[WHAT WILL HAPPEN TODAY]

Copilot will guide you through:
1. A friendly onboarding conversation  
2. A few simple questions about how you learn  
3. A check on your tools (IDE, OS, Python version)  
4. A choice:  
   - Use past Copilot experience  
   - OR start completely fresh  
5. A quick check on time limits and pacing  
6. Your very first Python chunk  

Everything is slow, gentle, and beginner-friendly.

============================================================
[DAY 01 ONBOARDING QUESTIONS]

Copilot will ask you:

1. What name should I use for you?  
2. How do you like to learn?  
3. Any pacing notes?  
4. Comfort level (1–3)?  
5. What editor/IDE are you using?  
6. What OS and Python version do you have?  
7. **Should I use your past Copilot experience or start fresh?**  
8. Do you have any time limits today?  
9. What kind of flow do you want today?  
10. Anything you want to review before we begin?

Just answer normally — no special format needed.

============================================================
[HOW THE LEARNING LOOP WORKS]

Copilot teaches using the Little Boxes Learning Loop:

- One small chunk at a time  
- You answer  
- Copilot evaluates  
- Copilot awards XP  
- Copilot adjusts difficulty  
- Then gives the next chunk  

This keeps learning:
- manageable  
- predictable  
- low-stress  
- personalized  

============================================================

Your first chunk will be:
- A tiny piece of code  
- A simple question  
- No pressure  
- No trick questions  
- No assumptions  

You’ll learn:
- What print() does

- How to edit your first line of code  

============================================================
[HOW TO SUBMIT CODE]

Before sending code to Copilot:
1. Run it in your IDE  
2. Fix any syntax errors  
3. If you can’t fix it, send the broken code anyway  
4. Copilot will help you debug  

Mistakes are expected.
Mistakes are part of the system.
Mistakes give XP.

- You can continue or stop
